DNA Passport 研究报告：实物NFT与DID赛道的进击者

要点

  DID 是 “去中心化身份 “（Decentralized Identity）的简称，它作为一种无需中心化机构验证的数字身份，重新给了用户掌握自身权利的机会。目前，DID 主要解决的问题包括所有权（用户的数据和身份管理权）问题以及原生 Web3 身份的缺位。针对以上两个问题，已经有多个项目基于不同场景的用户需求从多个方向尝试进行解决，大致可分为链上身份认证、链下身份认证类和链上+链下身份认证类。然而 DID 发展仍处于萌芽期，目前尚未有 DID 体系能积累起一定的网络规模效应，因此这条赛道也格外备受关注。 DNA DAO 将于 2023年年底推出其 DID 产品——DNA Passport ，继 AI智能显微扫描器推出后，DNA DAO 旗下实物版权系统也将从中心化平台进化为去中心化协议。那么DNA Passport 到底解决了什么问题？DNA Passport 和最新的AI智能显微扫描器有何关联呢？DNA DAO 为何要推出DNA Passport？此外，以 Bright ID、POAP 、Gitcoin Passport和 Galxe Passport 为代表的 DID 赛道正在如火如荼地开展，DNA Passport 和他们之间又有什么差异呢？DNA Passport 又将给行业带来哪些影响？正是在这系列问题的驱使下，笔者将对这些问题给出自己的研究分析。

背景介绍

  DNA是诞生于 2019年 4月的实物鉴权确权平台，起初的目的是为了更好的解决区块链落地实体。由于当时技术限制，大部分区块链与实体连接的方式都是在产品上贴一个二维码或者是芯片，这种区块链与实体的连接，这种溯源方式的易伪造性导致很多媒体认为区块链鉴证溯源是个伪命题。在当时程昔恩博士已经在区块链领域研究多年，在对当时区块链现状落地情况进行分析后，认为区块链溯源还是一个待解决的科学问题。便开始对其进行研究。DNA DAO于2018年起自主研发AI智能显微扫描器，研发过程攻克了三个关键难题：定位问题、提取问题和识别问题等原理研究和验证，利用一体式显微设备采集实物唯一性标识，同时完成了通过区块链存证艺术品身份信息，实现了信息的不可篡改、可追溯。截止 2023年 11 月，DNA DAO已经签约合作了两家机构，南昌市博物馆、中国海关。其已经帮助无数艺术品、古董鉴权确权，并多次获得国家颁发的奖项如：科创中国创新奖、创客中国优胜奖等等10多项奖。

DNA Passport
DNA Passport 介绍：

  DNA Passport 是一个建立在 Ceramic Network 上的线上+线下身份聚合 dApp，用户可以从Web2 和 web3 身份验证器中收集各种称为邮票的身份证明；邮票由各种 Web2 和 Web3 身份验证器提供，包括 Google、Facebook、Bright ID、ENS 和 Proof of Humanity。Passport 聚合邮票并根据线下用户提供实名认证信息汇总上链，例如，李雷通过链下提供身份信息并通过绑定线上的各种身份验证器，将各种信息汇总通过自己钱包上链。用户只需要通过地址就可知道该地址所对应的用户的信息。

DNA Passport 特点

信息自主：

  通过 DID 和 VC 等开放网络标准进行验证，它包含个人身份信息、WEB2账户信息以及WEB3信息。用户 Passport 中所有的信息都是由用户自行决定修改、增加或删除。

跨链验证：

跨多个区块链验证数据以获得可组合、可互操作的身份。这意味着用户只需持有一个 DNA Passport 就等于拥有了 Web3 网络通行证。

DNA Passport 技术实现

   DNA Passport 是基于 Ceramic 区块链网络开发的身份验证应用程序。Ceramic 是一个链下主权数据网络，将去中心化标识符 (DID) 映射到用户控制的数据流中。Ceramic 上的数据是公开的、无需许可且可验证的，从而解锁了网络上所有平台和服务之间的信息访问和互操作性。

  DNA Passport SDK 由一组分布在 NPM 上的数据库组成，以帮助开发人员与上的 Ceramic Passport 数据进行交互。

@DNA/passport-sdk-writer – 在 Ceramic 中写入经过身份验证的 DIDs Passport 流。

@DNA/passport-sdk-reader – 从任何 Passport 流中进行读取（在 Ceramic 上）。

@DNA/passport-sdk-verifier – 验证护照的内容。

@DNA/passport-sdk-types – 共享类型。

Ceramic 上的数据任何人都可以读取，知道用户的区块链地址、 DID 或者 Ceramic Stream ID 就可以从 Ceramic 获取数据，并且人们可以读取到用户的 Passport 信息，但是只有用户本人才可以往 Passport 里写入数据。当用户使用钱包签署一条消息以授予用户的访问权限时（例如当用户连接到 DNA Passport 应用程序），用户才会授予应用程序访问权限。Passport 需要用户唯一的钱包签名才能允许写入操作，只有想要写入用户PassPort的应用程序才需要请求用户签署这些消息。

DNA DAO 为何要推出 Passport
所有权问题

  由于互联网没有给用户提供原生的身份层，因此变成了由网站和应用来发放数字身份。筒仓式方式也许适用于早期互联网，然而现在的网民数量已达数十亿，筒仓式方法的弊端日益凸显。创建用户名和密码越来越成为身份证明的主流范式，尽管它越来越被认为是一种不安全的模型；而且普通人不得不管理越来越多的密码，这样也必然会带来糟糕的用户体验。更重要的是，用户实际上不拥有线上身份的所有权，而是从公司或中心化实体租用账户。因此线上身份容易着面临非法侵入、操控、审查的风险，或是易于丢失。

原生 Web3.0 身份的缺位

  「现如今，Web3.0 更多的是表达可转让的、金融化的资产，而不是对社会信任关系的一种编码。然而，许多核心经济活动，如无抵押贷款和建立个人品牌，都建立在持久的、不可转让的关系上。在不到十年的时间里，Web3.0 打造了一个前所未有的独特且灵活的平行金融系统，为金融交易带来了一个复杂和开放的生态系统，然后，金融交易的经济价值是由人类和他们的关系产生的。由于 Web3 缺乏代表这种社会身份的基础元素，它从根本上依赖于它所要超越的中心化的 Web2 结构，因此复制了其局限性。」——《去中心化社会：寻找 web3 的灵魂》

DNA Passport与其竞品的区别与联系

  针对以上两个问题，已经有多个项目基于不同场景的用户需求从多个方向尝试进行解决，大致可分为链上身份认证、链下身份认证类和链上+链下身份认证类。

-链上身份以 POAP 为代表，其通过给出席活动的人（账号）一个不可转让的 NFT 以证明一个人的身份。

-链下身份以 Bright ID 为代表，其通过真人参加 Zoom 线上视频会议进行真人认证，并且需要添加 2 个以上的 Bright ID 好友，用以将来的恢复身份。

-链上身份+链下身份以 DNA PassPort、 Gitcoin Passport 和Galax Passport 为代表。

DNA PassPort、 Gitcoin Passport 和Galax Passport

  DNA之于gitcoin最大不同点在于身份信息是否公开，用户在gitcoin上完成身份信息的上传之后gitcoin会给用户评分，它只会将分数上链，依旧无法解决用户参与核心的社会活动或建立个人品牌所遇到的匿名性问题

  DNA之于Galax最大不同点在于多维度信息聚合以及基于声誉的多中心链下信息验证，二者都采用了聚合链上、链下的身份信息；链下的 KYC 验证。不同之处在于DNA采用更多维度信息，用户可以自主决定上传哪些信息，非强制要上传某些信息！链下的 KYC 验证更是采用多个基于声誉的多中心服务商进行核验，保证用户信息的正确和安全。

DNA Passport给行业带来的影响
开发者

DNA Passport 是建立在 ERC-721 和 W3C DID/VC 标准（即将推出）等 Web 标准之上。使用 DNA Passport 的 JS SDK + Solidity SDK，开发人员可以在自己的平台上使用 Passport，用一行代码验证一个账户的具体实名信息。

用户

使用 DNA Passport，用户可以收集 “邮票”，从而帮助其他人更好的了解自己。当用户收集的“邮票”越多时，其它用户获取到的向外展示的信息就越多，将会更有利于用户参与线上核心经济活动。他们会在其他使用 DNA Passport 的杰出的 dApp 中受益。

传统身份解决方案商

  从短期来说，整个 DID（去中心化数字身份）赛道都还处于探索期，暂时构成不了威胁。但从长期来说，整个 DID 赛道的发展与 Web3 应用的普及是相辅相成的，难以想象一个非加密用户会平白无故弄一个去中心化身份，因为这对他来说并无用处。随着 Web3 应用逐渐渗透到普通民众的生活中，笔者相信 DID 也将逐渐取代传统身份解决方案商的市场。而对于DNA Passport 而言，在未来铸造NFT声明出处或金融借贷等应用场景迭代中进步而这种经验也将带给它的生态合作伙伴例如 Bright ID 带来帮助，与之共同成长，为用户提供更好的互联网身份。

一切运行顺利的话，DNA Passport 将形成这样的网络效应：

1. 更多的用户使用更多的 dApps 创造了更多的实用程序（“铸造NFT声明出处，查询NFT出处”）。 

2. 更多的 “邮票” 为集成PassPort的用户/dApps 创造更多的实用性（“邮票”/铸造NFT）。 

3. 更多的 dApp 创造了更多的实用程序（“邮票”/铸造NFT），从而吸引了更多的用户。
